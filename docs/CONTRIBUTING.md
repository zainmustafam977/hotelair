# Contributing to HotelAir

Thank you for your interest in contributing to HotelAir! This guide will help you get started.

## Code of Conduct

Please be respectful and constructive in all interactions with the community.

## Getting Started

1. Fork the repository
2. Clone your fork: `git clone https://github.com/YOUR_USERNAME/hotelair.git`
3. Create a feature branch: `git checkout -b feature/your-feature-name`
4. Make your changes
5. Test thoroughly
6. Commit with clear messages: `git commit -m "Add feature: description"`
7. Push to your fork: `git push origin feature/your-feature-name`
8. Open a Pull Request

## Development Workflow

### Before You Start
- Check existing issues and pull requests
- Discuss major changes in an issue first
- Follow the existing code style
- Install all development dependencies: `npm install`

### Making Changes
1. Keep commits atomic and focused
2. Write clear commit messages
3. Add comments for complex logic
4. Update documentation as needed
5. Test your changes: `npm test`

### Code Style
- Use consistent indentation (2 spaces)
- Use meaningful variable names
- Follow JavaScript best practices
- Use async/await over callbacks
- Add JSDoc comments for functions

### Testing
```bash
# Run all tests
npm test

# Run tests in watch mode
npm test -- --watch

# Run tests with coverage
npm test -- --coverage
```

### Before Submitting a PR
- [ ] Code follows style guidelines
- [ ] All tests pass
- [ ] New features have tests
- [ ] Documentation is updated
- [ ] No console errors/warnings
- [ ] Commits are squashed if needed

## Pull Request Process

1. Update documentation with any new features
2. Increase version numbers following semver
3. Describe changes clearly in PR description
4. Link related issues with "Closes #123"
5. Request review from maintainers
6. Address review feedback promptly
7. Squash commits if requested

## Types of Contributions

### Bug Reports
Include:
- Clear description of the bug
- Steps to reproduce
- Expected vs actual behavior
- Screenshots/logs if applicable
- Your environment details

### Feature Requests
Include:
- Clear use case and benefit
- Proposed API/UI if applicable
- Any alternative approaches considered

### Documentation
- Improve existing docs
- Add missing documentation
- Fix typos and clarity issues
